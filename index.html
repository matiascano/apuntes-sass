Unos conceptos básicos de Sass

Qué es Sass?

Sass es un preprocesador de CSS que mejora tu productividad al escribir código más limpio, modular y reutilizable. A continuación, explorarás los conceptos básicos de Sass, cómo usarlos y cuándo son útiles.

Instalación

Para comenzar a usar Sass, necesitas seguir los siguientes pasos:

  1. Instalamos usando Node.js: npm install -g sass
  2. Creamos una carpeta sass dentro del proyecto, la cual contendrá los archivos de Sass
  3. Dentreo de la carpeta creamos un archivo.scss (scss es la extensión para sass)
  4. En la terminal, navegamos a la carpeta del proyecto y ejecutamos el comando: sass --watch sass:css o sass --watch ./sass/archivo.scss ./css/archivo.css
  5. Esto le indica a Sass que observe los cambios en la carpeta sass y compile automáticamente los archivos Sass a CSS en la carpeta css.
  6. Si no tienes Node.js instalado, puedes descargarlo desde nodejs.org.

Una vez instalado, puedes compilar archivos Sass con el comando:

          sass input.scss output.css

Esto convertirá tu archivo Sass en un archivo CSS estándar que los navegadores pueden entender.

Esta ultima linea fundamental, ya que Sass no es un lenguaje final, sino un preprocesador que transforma tu código Sass en CSS.

:variables

Permiten almacenar valores reutilizables como colores, fuentes o tamaños.

Versión Sass:


$primary-color: #1daa7b;

.button {
  background-color: $primary-color;
}

Este código define una variable `$primary-color` y la usa dentro de una clase `.button`. Esto evita repetir el color varias veces y facilita su mantenimiento.

Resultado CSS:


.button {
  background-color: #1daa7b;
}

Cuándo usarlo: Para colores de marca, breakpoints, tipografías, o cualquier valor que se repita a lo largo del proyecto.

Anidamiento (Nesting)

Organiza selectores CSS de forma visual y semántica.

Versión Sass:


nav {
  ul {
    list-style: none;
    li {
      display: inline-block;
    }
  }
}

Este ejemplo muestra cómo anidar reglas CSS. Es útil para mantener el estilo relacionado con estructuras HTML jerárquicas.

Resultado CSS:


nav ul {
  list-style: none;
}
nav ul li {
  display: inline-block;
}

Cuándo usarlo: Para elementos como menús, formularios o componentes complejos donde la relación padre-hijo sea clara.

Mixins

Bloques reutilizables que pueden recibir parámetros.

Versión Sass:


@mixin btn($bg) {
  padding: 10px 20px;
  background: $bg;
  color: white;
}

.primary-btn {
  @include btn(#1daa7b);
}

Los mixins permiten reutilizar bloques de código con variaciones. En este ejemplo, se define un estilo de botón que recibe un color de fondo.

Resultado CSS:


.primary-btn {
  padding: 10px 20px;
  background: #1daa7b;
  color: white;
}

Cuándo usarlo: Para componentes reutilizables como botones, tarjetas, animaciones o media queries personalizados.

Funciones

Realiza cálculos personalizados y devuelve un valor.

Versión Sass:


$base: 16px;

@function double($value) {
  @return $value * 2;
}

h1 {
  font-size: double($base);
}

Las funciones permiten realizar operaciones dinámicas. Este ejemplo define una función `double()` que multiplica un valor por dos.

Resultado CSS:


h1 {
  font-size: 32px;
}

Cuándo usarlo: Para cálculos dinámicos como tamaños responsivos, espaciados, opacidad, etc.

Partials & Import

Divide tu código en archivos pequeños y organiza tus estilos.

Versión Sass:


// _variables.scss
$primary: #1daa7b;

// main.scss
@import 'variables';

body {
  color: $primary;
}

Los partials son archivos SCSS que comienzan con `_`, y no se compilan solos. Los puedes importar desde otros archivos para modularizar tu código.

Resultado CSS:


body {
  color: #1daa7b;
}

Cuándo usarlo: Para proyectos grandes, donde la organización y separación de responsabilidades es importante (ej: variables, reset, layout, components).

Extend / Herencia

Reutiliza bloques completos de estilo sin duplicar código.

Versión Sass:


%btn {
  padding: 10px 20px;
  border-radius: 5px;
}

.btn-success {
  @extend %btn;
  background: green;
}

Con `@extend`, puedes heredar estilos definidos en otro selector. Aquí, `.btn-success` hereda todo el estilo de `%btn` y agrega uno nuevo.

Resultado CSS:


.btn-success {
  padding: 10px 20px;
  border-radius: 5px;
  background: green;
}

Cuándo usarlo: Cuando tienes múltiples clases con estilos similares, como botones, alertas o mensajes.

@each

Genera múltiples reglas a partir de una lista.

Versión Sass:


$colors: primary red blue;

@each $color in $colors {
  .text-$color {
    color: $color;
  }
}

El bucle `@each` itera sobre una lista y genera estilos dinámicos. En este ejemplo, se crean tres clases de texto basadas en los colores definidos.

Resultado CSS:


.text-primary {
  color: primary;
}
.text-red {
  color: red;
}
.text-blue {
  color: blue;
}

Cuándo usarlo: Para generar automáticamente clases utilitarias, paletas de colores, iconos, etc.

@for

Itera un número específico de veces para crear estilos escalables.

Versión Sass:


@for $i from 1 through 5 {
  .col-#{$i} {
    width: 20px * $i;
  }
}

El bucle `@for` permite generar estilos basados en un rango numérico. Aquí creamos cinco columnas con anchos progresivos.

Resultado CSS:


.col-1 { width: 20px; }
.col-2 { width: 40px; }
.col-3 { width: 60px; }
.col-4 { width: 80px; }
.col-5 { width: 100px; }

Cuándo usarlo: Para generar sistemas de rejilla, espaciados, títulos responsivos u otros patrones numéricos.

if / else

Ejecuta estilos condicionalmente según una expresión lógica.

Versión Sass:


$theme: dark;

@if $theme == dark {
  body {
    background: black;
    color: white;
  }
} @else {
  body {
    background: white;
    color: black;
  }
}

Con `if` y `else`, puedes aplicar estilos dependiendo del valor de una variable. En este caso, cambiamos entre tema claro y oscuro.

Resultado CSS:


body {
  background: black;
  color: white;
}

Cuándo usarlo: Para configuraciones temáticas, estilos condicionales o generación de variantes dinámicas.
